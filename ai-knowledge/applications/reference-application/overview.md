# Reference Application Overview

This file demonstrates the expected structure for an application knowledge
pack. Replace the content in an adopting repository with the owning team's
actual application overview, business journeys, integration points, and support
runbook links.

## Key Journeys

- Capture a business request.
- Validate business rules.
- Persist transaction state.
- Publish downstream events.

## Support Notes

- Start investigation from the business reference ID.
- Compare application state, database state, and downstream event state.
- Prefer approved query catalogs and masked summaries for production evidence.
